# P0.5-001 Metrics Export For Lightweight Dashboard

Type: AFK

User stories covered: 5, 19, 20

## Parent

`docs/prd/Jetson_Physical_AI_PRD.md`

## What to build

Create a small export path that turns P0 metrics and health data into JSON or compact files suitable for a local dashboard, without making the dashboard part of P0 validation.

## Acceptance criteria

- [ ] Exporter reads or receives pipeline metrics, system metrics, and health state.
- [ ] Exported data includes health state, publish rate, receive rate, drop count, drop rate, avg latency, p95 latency, p99 latency, CPU, RAM, temperature, and GPU indicator where available.
- [ ] Export writes to project-local runtime results or serves localhost-only data.
- [ ] Export format is documented.
- [ ] Exporter does not expose secrets or company identifiers.
- [ ] P0 validation remains possible without this exporter.

## Blocked by

- P0-016

## Verification commands

- Exporter command or launch command.
- Inspect generated JSON or data file.
- Confirm output path is under `runtime/`.

## Runtime artifact location

`runtime/results/dashboard/`

## Cleanup and rollback

Delete only exported dashboard feed files under runtime.

## Out of scope

- Building the visual dashboard.
- Public network exposure.
