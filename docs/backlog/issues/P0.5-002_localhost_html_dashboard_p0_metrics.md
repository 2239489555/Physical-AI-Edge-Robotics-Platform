# P0.5-002 Localhost HTML Dashboard For P0 Metrics

Type: AFK

User stories covered: 5, 19, 20, 25

## Parent

`docs/prd/Jetson_Physical_AI_PRD.md`

## What to build

Build a lightweight HTML dashboard for local or forwarded viewing of P0 health and metrics. The dashboard is a display enhancement, not a P0 gate.

## Acceptance criteria

- [ ] Dashboard shows current health state.
- [ ] Dashboard shows publish rate, receive rate, p95 latency, p99 latency, drop count, and drop rate.
- [ ] Dashboard shows CPU, RAM, temperature, and GPU indicator where available.
- [ ] Dashboard reads the P0.5 metrics export feed.
- [ ] Dashboard binds to localhost by default or is static HTML opened locally.
- [ ] README documents access through local browser, external display, Cursor, or SSH port forwarding.
- [ ] Dashboard never becomes the only source of validation evidence.

## Blocked by

- P0.5-001

## Verification commands

- Dashboard start command or static file open instructions.
- Inspect dashboard with a browser.
- Confirm network binding is localhost by default if a server is used.

## Runtime artifact location

`runtime/results/dashboard/` for generated feed files.

## Cleanup and rollback

Stop any local dashboard process started by the project. Remove only runtime feed files.

## Out of scope

- Complex frontend framework.
- Authentication system.
- Public deployment.
